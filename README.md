# ZARRA RESORT — Telegram-бот (инструкция запуска на Mac)

Тебе не нужно ничего программировать. Просто повтори шаги по порядку.

## Что нужно один раз установить

**Python** — «движок», на котором работает бот.
1. Зайди на https://www.python.org/downloads/
2. Скачай кнопкой «Download Python 3.x» (macOS installer).
3. Открой скачанный файл и установи (Далее → Далее → Установить).

## Подготовка папки

1. Создай папку, например `zarra_bot` на Рабочем столе.
2. Положи в неё оба файла: `bot.py` и `requirements.txt`.

## Первый запуск (по шагам)

1. Открой программу **Терминал** (Spotlight: ⌘+Пробел → набери «Terminal»).
2. Напечатай `cd ` (буквы c, d и пробел), затем **перетащи папку `zarra_bot`** из Finder прямо в окно Терминала — путь подставится сам. Нажми Enter.
3. По очереди вставь команды (после каждой — Enter):

```
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

4. Запусти бота:

```
python bot.py
```

5. При первом запуске бот попросит **токен** (от BotFather) и **Gemini API ключ** — вставь их по очереди и нажми Enter. Он сохранит их и больше спрашивать не будет.

Когда увидишь строку «Бот запущен» — всё работает. ✅

## Как проверить

- Открой Telegram, найди своего бота по username (который придумал в BotFather), напиши ему «Привет» — он ответит.
- Либо подключи его к профилю: *Настройки → Автоматизация чатов* (для теста выбери «Only Selected Chats»).

## Полезное

- **Остановить бота:** в Терминале нажми `Ctrl + C`.
- **Запустить снова потом:** открой Терминал, перейди в папку (шаг 2), затем:
  ```
  source venv/bin/activate
  python bot.py
  ```
- **Чтобы Mac не «засыпал», пока бот работает** (например, на время демо), запускай так:
  ```
  caffeinate -i python bot.py
  ```
  И держи ноутбук открытым и подключённым к зарядке.

## Запуск 24/7 на сервере (Oracle)

Чтобы бот работал круглосуточно и сам поднимался после перезагрузки/сбоя,
на сервере используется systemd-сервис (файл `zarra-bot.service` в этой папке).

Один раз настроить на сервере:

```
# 1. Скопировать сервис в систему (поправь путь, если папка другая)
sudo cp /home/ubuntu/zarra_bot/zarra-bot.service /etc/systemd/system/

# 2. Включить автозапуск и стартовать
sudo systemctl daemon-reload
sudo systemctl enable --now zarra-bot
```

Полезные команды:

```
sudo systemctl status zarra-bot      # работает или нет
sudo systemctl restart zarra-bot     # перезапустить (например после правок bot.py)
journalctl -u zarra-bot -f           # смотреть логи вживую
```

⚠️ Перед первым `systemctl` в файле `zarra-bot.service` поправь `User`,
`WorkingDirectory` и `ExecStart` под своего пользователя и путь на сервере.

## Языки (русский / узбекский / английский)

- При первом `/start` бот показывает кнопки выбора языка и сам угадывает язык
  по настройкам Telegram гостя. Сменить язык в любой момент — команда `/lang`.
- Меню, кнопки и мастер брони переключаются на выбранный язык. ИИ и так
  отвечает на языке гостя.

## Администраторы и сводка

- Ежедневная утренняя сводка приходит владельцу и всем администраторам.
- Добавить администратора (например директора): отправь боту в личке
  `/add_admin @username`, затем попроси его открыть бота и нажать «Старт».
  Список — `/admins`, удалить — `/remove_admin @username`.
- Команды «для своих» видны в меню команд только владельцу и админам.

## Правки без кода (для владельца)

Меняются прямо в боте, без правки кода и деплоя:

- **Цены:** `/prices` — показать текущие; изменить —
  `/setprice comfort 1 будни 1,6 млн` (шале: comfort/lux; слот: 1 день, 2 ночь,
  3 полный; тип: будни/вых). Меняет цены в мастере, в «🏡 Шале и цены» и в
  ответах ИИ. ⚠️ Цены в мини-приложении лежат отдельно — их обновляем через
  правку `zarra_webapp` (скажи мне).
- **Объявление / акция:** `/announce Акция: в июле −10% в будни` — бот будет
  упоминать это в ответах и покажет в «🏡 Шале и цены». Убрать — `/announce off`.

## Рассылка по гостям

- `/broadcast <текст>` — отправить сообщение всем гостям, кто писал боту.
- Бот покажет предпросмотр и число получателей, отправит после подтверждения.

## Предоплата (карта + чек) — без овербукинга

- Один раз задай карту: в личке боту `/set_card 5614 6818 7306 2941 Anvar Bahronov`.
- Порядок (менеджер всегда проверяет Exely — авто-продажи НЕТ):
  1. Заявка пришла в группу.
  2. Проверил Exely. Свободно → жмёшь **«💳 Запросить предоплату»** под заявкой.
  3. Бот сам пишет гостю: сумму 50% (считает от цены) + карту + просьбу прислать чек.
  4. Гость присылает **скриншот чека** → бот кладёт его в группу под заявку.
  5. Сверил чек и Exely → жмёшь **«✅ Подтвердить»** (тогда дата занимается).
- **Таймер:** у гостя есть ~3 часа на оплату (поле `pay_hours` в базе). Не успел —
  бот снимает запрос, пишет гостю и сообщает в группу «слот свободен».

## Лист ожидания

- Дата занята (проверил в Exely) → жмёшь **«⏳ В лист ожидания»** под заявкой из
  мастера. Бот пишет гостю, что поставил его в очередь.
- Когда место освобождается (отклонил бронь или `/free`) — бот **сам уведомляет**
  ждущих гостей на эту дату.

## AI-планировщик мероприятий

- Кнопка «🎉 Мероприятие» (или `/plan`) — гость описывает событие (повод, число
  гостей, дата, пожелания), а бот подбирает шале, слот, считает примерную смету
  и предлагает блюда. Работает на всех 3 языках.

## Голосовые ответы

- Если гость пишет боту **голосовым** — бот отвечает и текстом, и **голосом**
  (озвучка через gTTS, на языке гостя). Нужен пакет `gTTS` (в requirements.txt).
- Для «голосового пузыря» (а не аудиофайла) на сервере желателен ffmpeg:
  `sudo apt install -y ffmpeg`. Без него бот пришлёт озвучку как аудио — тоже работает.

## График загрузки

- `/chart` (для своих) — присылает картинкой график обращений и заявок за 14 дней
  плюс итоги за 7 дней (обращения, заявки, подтверждено, конверсия).

## Панель владельца (дашборд)

- `/dashboard` (для своих) — брендовая картинка с ключевыми метриками за 30 дней:
  выручка (оценка по ценам), подтверждено, заявок, обращений, конверсия, средний
  чек, рейтинг, топ-шале, топ-источник + итог за 7 дней. Плюс график динамики.

## AI-ревеню-менеджер

- `/revenue` (для своих) — ИИ анализирует спрос, конверсию, цены и загрузку за
  30 дней и даёт 4–6 конкретных рекомендаций (что поднять/снизить, где акция).

## Персонализация (CRM)

- Бот запоминает гостей (имя, телефон, что брали). Вернувшегося встречает лично:
  «С возвращением, {имя}! В прошлый раз брали {шале} — повторить?». ИИ тоже
  учитывает профиль и обращается по имени.

## Карточка-подтверждение (картинка гостю)

- Когда сотрудник жмёт «✅ Подтвердить», гость получает красивую брендовую
  картинку «Бронь подтверждена» (шале, дата, слот, гости, имя).
- Нужен пакет Pillow (в `requirements.txt`) и шрифты в папке `fonts/` (уже в
  репозитории). Если Pillow не установлен — бот работает как обычно, просто без
  картинки.

## Мини-приложение (Telegram Web App)

Красивое приложение бронирования (файл `../zarra_webapp/index.html`).

1. Размести папку `zarra_webapp` на любом HTTPS-хостинге (Netlify: перетащи
   папку в окно — получишь ссылку вида `https://…netlify.app`).
2. Дай боту эту ссылку — в личке владельцем: `/set_webapp https://…netlify.app`
   (убрать кнопку: `/set_webapp off`).
3. В меню бота появится кнопка «🚀 Забронировать в приложении». Гость заполняет
   заявку в приложении → она падает в группу заявок, как обычный мастер.

Занятость гостю в приложении не показывается; пересечения видит сотрудник в
карточке. BotFather настраивать не нужно (кнопка работает через меню-клавиатуру).

## Бот общается в группе

- В группе бот отвечает как дружелюбный **помощник** резорта, когда к нему **обратились**:
  упомянули через `@ник_бота` или **ответили (reply)** на его сообщение.
- На обычную переписку он НЕ реагирует — только когда позвали. Лимит, чтобы
  не флудить. Бронированием в чате не занимается (зовёт написать боту в личку).
- Менять характер можно в блоке `GROUP_RULES` в `bot.py`.

### Чтобы бот был «в курсе беседы» (помнил, о чём говорили)
По умолчанию (приватность включена) бот видит ТОЛЬКО сообщения, где его упомянули
или ответили ему — поэтому общий разговор он не запоминает. Чтобы он следил за
беседой и отвечал «по моменту»:
1. @BotFather → `/mybots` → выбрать бота → Bot Settings → Group Privacy → **Turn off**
   (или `/setprivacy` → выбрать бота → **Disable**).
2. ВАЖНО: после этого **удали бота из группы и добавь заново** — Telegram применяет
   смену приватности только при повторном добавлении.

После этого бот тихо запоминает последние ~30 реплик чата (`GROUP_MEMORY` в `bot.py`)
и, когда к нему обратятся, ответит с учётом контекста. Отвечает он всё равно только
когда позвали — расходы не растут. Память хранится до перезапуска бота.

## Живой чат с гостем (через группу заявок)

- Под каждой заявкой и алертом «нужен человек» можно просто **ответить reply**
  прямо в группе — бот перешлёт твой текст гостю.
- После этого сообщения гостя приходят в группу, а ты отвечаешь reply.
  Когда закончил — нажми «🔚 Завершить диалог», и дальше снова отвечает бот.

## Бронирование кнопкой

- Кнопка «📝 Забронировать» (или `/book`) ведёт гостя по шагам:
  **Шале → Дата → Слот → гости → имя → телефон → пожелания → подтверждение** —
  и в конце шлёт заявку в группу, как обычный сбор через ИИ.
- Дата выбирается из календаря. **Загрузка гостю НЕ показывается** — все даты
  выглядят одинаково доступными, чтобы конкуренты не видели занятость.
- Так как дата выбрана до слота, на кнопках тарифа показывается СРАЗУ точная
  цена (будни или +20% выходного), а не обе.

## Календарь занятости (только для сотрудников)

- Считается КОЛИЧЕСТВО вилл: Комфорт — 9, Люкс — 3. На одну дату+слот можно
  взять столько броней, сколько вилл. Дата «занята» только когда заняты ВСЕ.
- Слоты по времени: день (С1) и ночь (С2) — независимы; полный день (С3)
  занимает виллу и днём, и ночью.
- Загрузка нигде НЕ видна гостю. Календарь видят только сотрудники — команда
  `/busy` (можно прямо в группе заявок): показывает по датам, сколько вилл
  свободно «день / ночь» для Комфорта и Люкса.
- Когда сотрудник жмёт «✅ Подтвердить» под заявкой из мастера — занимается одна
  вилла (и показывается, сколько осталось). «❌ Отклонить» — освобождает её.
- Если гость выбрал дату, где вилл уже не осталось — он этого НЕ видит и спокойно
  отправляет заявку; а в карточке сотруднику показывается «⚠️ возможно пересечение».
- Вручную: `/block lux 2026-07-05 3` (занять одну виллу), `/free …` (освободить).
- Команды для своих:
  - `/busy` — список занятых дат впереди (или `/busy lux`).
  - `/block lux 2026-07-05 3` — занять вручную (шале: lux/comfort; слот: 1 день,
    2 ночь, 3 полный день).
  - `/free lux 2026-07-05 3` — освободить.

## Источники гостей (откуда пришли)

- Делай ссылки с меткой источника:
  `https://t.me/ИМЯ_БОТА?start=instagram` (или `?start=qr`, `?start=tiktok`…).
  Метку (после `start=`) бот запоминает за гостем.
- В `/report` показывается разбивка заявок по источникам — видно, какой канал
  приносит брони.

## Напоминания и отзывы (автоматически)

- За день до заезда бот напоминает гостю о брони.
- На следующий день после выезда просит оценить отдых (⭐1–5). Высокая оценка —
  мягко зовём оставить отзыв в Instagram; низкая — прилетает алерт в группу.
- Работает для подтверждённых броней, оформленных через мастер (с точной датой).

## Данные и бэкапы

- Заявки, галереи и настройки хранятся в `media_store.json` (создаётся сам).
- Раз в сутки бот делает копию в папку `backups/` (хранит 14 последних дней).
- ⚠️ Если переносишь бота — НЕ копируй пустой `media_store.json` с компьютера
  на сервер поверх рабочего: перезатрёшь заявки и галереи. База живёт на сервере.

## Важно

- Токен и ключ хранятся в файле `.env` в этой папке. Никому не показывай эту папку
  и не выкладывай `.env`, `*.key`, `media_store.json` в открытый доступ
  (они уже занесены в `.gitignore`).
